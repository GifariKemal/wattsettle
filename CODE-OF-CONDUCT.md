# 🌱 Code Of Conduct

## Our Standard

We keep collaboration direct, technical, and respectful.

| Expected | Not acceptable |
|---|---|
| Clear feedback | Personal attacks |
| Evidence based review | Vague blame |
| Small focused changes | Hidden rewrites |
| Respect for ownership | Reverting unrelated work |
| Security first mindset | Publishing secrets |

## 🙂 Good Collaboration

- Be specific about the file, behavior, and risk.
- Prefer reproducible examples.
- Keep comments focused on the work.
- Assume changes may affect a live pitch.

## Enforcement

Project maintainers may remove comments, issues, or contributions that harm the project or the people working on it.

For conduct concerns, contact:

```txt
security@suriota.id
```
